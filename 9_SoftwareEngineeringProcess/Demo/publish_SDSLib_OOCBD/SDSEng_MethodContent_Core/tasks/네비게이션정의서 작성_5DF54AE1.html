      Main Description

      수행 가이드(Task Guideline)

      1. 한 화면의 작업을 마치기 위해 여러 개의 팝업 화면이 필요한 경우 이러한 내용을 네비게이션 정의에 어떻게 표현할 것인지 정의해야 한다. 예컨대 회원가입 화면의 경우 최종적으로 ‘가입’ 버튼을 클릭하기 전에 우편번호 검색과 같은 팝업 화면을 여러 개 사용하는데 이러한 내용을 네비게이션 정의에 모두 표현할 것인지, 표현한다면 어떤 방법으로 표현할 것인지 정의해야 한다.
      2. 효과적인 네비게이션은 사용자의 현재 위치가 어디이며, 앞으로 업무를 수행하기 위해서는 어디로 가야 하는지, 어떻게 가야 하는지를 직관적으로 알려 주어야 한다.
      3. 개별 시스템마다 네비게이션이 다르게 구성되기 보다는 동일한 네비게이션 방식을 지니는 메뉴(Menu)로 구성되어야 한다. 로컬 네비게이션의 메뉴 체계를 설계할 때는 메뉴의 너비(Breadth)와 깊이(Depth)의 문제가 사용성을 결정짓는 주요한 원인이다. (일반적으로 메뉴 구조의 너비가 넓어 질 때 보다는 깊이가 깊어 질 때 메뉴의 사용성이 떨어진다고 보고하고 있다)
      4. 네비게이션 시스템에서 거쳐야 할 단계에 따른 클릭 수가 많아지게 되면 업무시스템에서 효율이 저하될 수 있다. 그러므로 가능한 최소한의 단계를 제공할 수 있도록 해야 한다.
      5. 사이트 맵이나 검색 시스템과 같은 대안을 함께 제공해야 한다.
      6. 선택 페이지에 대한 설명을 제공해야 한다. 예를 들어 선택한 페이지나 창의 제목이 명확하게 제공되어야 한다. 페이지 제목, 버튼, 메뉴 그룹의 레이블은 쉽게 이해 할 수 있어야 한다. 제시되는 페이지 내역 중 어떤 것이 사용 가능한 것인가 보여줄 수 있어야 한다.
      7. 동일한 화면으로 이동하는 경로가 둘 이상일 경우 메뉴에 의한 경로는 하나로 제한하며, 그 밖의 경로는 화면 내에서 별도 링크(관련정보 등)를 사용하여 연결한다.
      Steps
      분석자는 화면프로토타입과 유스케이스정의서를 바탕으로 각 화면에서 일어나는 이벤트를 근거로 다음 화면을 식별한다.
      분석자는 각 화면에서 다음 화면으로 가기 위한 이벤트, 조건, 화면 전환시 전달되는 데이터 등을 기재한다.
