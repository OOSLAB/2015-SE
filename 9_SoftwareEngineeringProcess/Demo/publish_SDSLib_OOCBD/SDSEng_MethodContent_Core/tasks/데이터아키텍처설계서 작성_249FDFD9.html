Task: 데이터아키텍처설계서 작성
Disciplines: 설계 아키텍처모델
Purpose
데이터 아키텍처의 구조를 정의하고, 데이터 아키텍처를 구현하기 위해 필요한 구성요소 및 설계 매커니즘들에 대해 상세화하여 설계한다.

    • 데이터아키텍처설계서
      Process Usage
      • 아키텍처 설계  >  데이터아키텍처설계서 작성
      Main Description

      수행 가이드(Task Guideline)

      1. 데이터 아키텍처 뷰의 Layer와 구성요소는 프로젝트 구축 범위 및 특성을 고려하여 정의하도록 한다.
      2. 설계 매커니즘은 시스템적으로 동일한 문제에 대한 해결방안을 제시하는 것으로, 프로젝트 특성에 따라 전체적으로 데이터 설계 이슈에 대한 항목들을 식별하여 해당 이슈에 대한 방안들을 기술하도록 한다.
      3. 설계 매커니즘을 정의할 때 타 아키텍처 영역과 관련성이 높은 부분들에 대해 SA,TA들과 협업하여 매커니즘을 정의하도록 한다. 예) 데이터 보안 등
      Steps
      데이터 아키텍처에 대한 데이터 아키텍처 뷰를 정의한다.
      • 데이터 아키텍처 Layer 및 세부 구성요소를 정의하여 데이터구성도를 작성하고, Layer별 역할을 정의한다.
      • 데이터 아키텍처 Layer 간 세부 데이터 흐름요소를 식별하고, 각 흐름에 대한 설계방안을 정의한다.
      • 데이터 통합 및 분산 구성요소를 식별하고, 각 구성에 대한 설계방안을 정의한다.(선택)
      • 데이터 아키텍처에 대한 관리측면의 세부 구성요소를 식별하고, 각 관리요소에 대한 설계방안을 정의한다.(선택)
      데이터 아키텍처를 위한 설계 매커니즘을 정의한다.
      • 데이터 아키텍처를 구현하기 위한 설계 관점에서 상세화할 설계 매커니즘을 식별한다.
      • 식별된 설계 매커니즘 별 설계방안을 정의한다. 데이터 아키텍처 설계 매커니즘은 다음과 같은 항목 등이 포함될 수 있다.
        • 데이터베이스 설계 방안
        • 데이터 표준화 방안
        • 데이터 보안 방안
      • 기술인프라 및 어플리케이션 측면을 고려하여 데이터 구성요소의 배치와 설계 매커니즘의 정의를 지원한다.
      • 데이터 아키텍처 요소 별 설계방안을 통합하여 정합성을 검토한다.
      • 타 아키텍처와의 상호 정합성을 검토한다.
