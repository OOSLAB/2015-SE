Task: 컨버전된데이터 생성
Disciplines: 컨버전및릴리이즈모델
Purpose
구축된 데이터베이스에 운영에 필요한 데이터를 컨버전 프로그램 등을 통해서 로드한다.

      Process Usage
      • 컨버전 및 릴리이즈  >  컨버전된데이터 생성
      Main Description
      수행 가이드(Task Guideline)
      1. 컨버전 대상이 존재하지 않는 경우 해당 태스크는 생략될 수 있다.
      2. 컨버전 방법은 프로그램 혹은 툴을 이용할 수 있다.
      3. 컨버전 대상이 되는 데이터를 컨버전하기 위해 필요한 정보는 다음과 같다.
        • 구 컨버전 대상(데이터)의 위치
        • 각각의 컨버전 대상을 컨버전하는데 필요한 프로그램명
      5. 컨버전 결과에 대한 정확성 검증 시 반드시 고객이 참여하도록 한다.
      6. 오류 발생시 컨버전 계획서에 정의된 처리방법에 따라 조치한다.
      7. 오류 발생시 이전 상태로 데이터를 복구하기 위해 컨버전 프로그램 수행 이전에 반드시 구 데이터를 백업받도록 한다.
      8. 컨버전 결과에 대한 정확성 확인 후 컨버전된 최초 데이터는 백업하여 향후 오류 발생시 복구 가능하도록 한다.
      Steps
      컨버전 설계에서 도출된 컨버전 대상 데이터의 목록, 범위, 방법을 확인한다.
      컨버전 실시전의 구 데이터는 데이터를 백업한다.
      컨버전에 필요한 컨버전용 프로그램을 수행한다.
      컨버전된 데이터를 신 시스템으로 이관한다.
      컨버전 결과에 대한 정확성을 확인한다.
