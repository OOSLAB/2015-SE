Description
Main Description

작성항목

  1. 프로그램ID : 배기기능을 처리하기 위한 프로그램ID를 기재한다.
  2. 관련 배치ID: 해당 배치프로그램과 관련된 배치ID를 기재한다. 여러 배치 기능에서 공통적으로 사용하는 서브 모듈 등은 관련 배치ID가 여러 개 있을 수 있다.
  3. 프로그램 개요: 해당 배치 프로그램의 역할이 나타나도록 개략적인 설명을 기술한다.
  4. I/O 명: 배치 프로그램 수행과 관련된 입출력데이터를 명시한다. 입출력 데이터를 명시할 때는 데이터타입도 같이 명시한다.
    • 입력데이터명: 배치 프로그램 수행 시에 필요한 입력데이터명을 기재한다.
    • 출력데이터명: 배치가 수행되고 난 후 만들어지는 출력데이터를 기재한다.
  6. 처리로직: 배치 프로그램을 구현하기 위한 처리 로직을 기술한다.

Key Considerations

작성시 고려사항

  • 배치프로그램명세서는 배치 기능을 처리하기 위해 구성한 프로그램 단위로 작성한다. 배치 기능 단위로 작성할 경우, 여러 배치 기능에 공통적으로 사용되는 서브 모듈은 처리로직이 중복기술될 수 있으며, 배치기능의 복잡도에 따라 프로그래밍언어가 혼합되어 사용될 수 있으므로(예: Java와 Stored Procedure) 프로그램 단위로 작성하고, 프로그램간의 사용관계를 처리로직상에 나타나도록 기술한다.
  • 배치 프로그램의 입력데이터 및 출력데이터는 데이터타입도 함께 기재한다. 일반적인 파라미터인 경우에는 해당 데이터타입(해당 개발언어에 따른 데이터타입)을 명시하고 그렇지 않은 경우에는 파일형태인지 테이블형태인지를 명시하도록 한다. 파일형태인 경우 SAM파일 혹은 XML파일 형태 등 다양할 수 있다.
  • 입출력데이터가 파일이나 임시테이블 형태인 경우는 각 필드정보(길이 및 데이터타입 정보포함)를 비고항목 부분에 기술할 수 있다.
  • 처리로직은 프로그램 구현이 가능한 수준으로 작성한다. 처리순서대로 해당 스텝에서 필요한 데이터정보 및 사용 모듈 등이 나타나도록 기술한다.
