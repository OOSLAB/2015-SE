Description
Main Description

작성항목

  1. (TO-BE)업무구분 : 업무명 또는 서브시스템명
  2. (TO-BE)테이블명 : 테이블이름
  3. (TO-BE)테이블ID : 해당 테이블의 물리적인 ID
  4. (AS-IS)업무구분: 업무명 또는 서브시스템명
  5. (TO-BE)테이블명 : 해당 테이블 또는 데이터의 이름
  6. (TO-BE)테이블ID : 해당 테이블 또는 데이터의 물리적인 ID
  7. 매핑룰 : 테이블 매핑과 관련한 특이사항을 기술한다. 예를 들어 기존 시스템과 동일한 레이아웃 이거나, 엑셀 등의 DB화 되지 않은 자료를 이용하거나, 별도의 입력 프로그램을 통하여 데이터를 생성해야 하는 경우 등에 명시한다. 특히, 기존 시스템과 신 시스템의 데이터가 M:N 관계인 경우에는 컨버전 로직을 작성하고, 컨버전과 관련된 제약사항이 있다면 이에 대하여 기술한다.
  8. 비고 : 시스템 또는 서브시스템 단위의 컨버전 관련 특이사항을 기술한다. 또는 기존 시스템의 테이블 중 컨버전 대상에서 제외되는 경우에 내용을 기술한다.

Key Considerations

작성시 고려사항

  • 테이블 매핑은 신 시스템의 테이블을 기준으로 작성하며, 데이터 컨버전 또는 입력 프로그램을 통하여 데이터를 구축하는 모든 테이블을 명시한다. 작성 후에는 기존 시스템의 테이블을 기준으로 매핑여부를 확인하여 누락된 테이블에 대한 검증을 하여야 한다.
  • 매핑이 불가능한 테이블에 대해서는 고객과 협의된 자료구축방안을 비고란에 기술한다.
  • 신 시스템의 테이블 하나에 하나 이상의 기존 시스템의 테이블이 매핑되는 경우에는 (TO-BE)테이블에 해당 테이블을 모두 명기한다.
  • 기존 시스템의 테이블 하나가 신 시스템의 여러 테이블로 매핑되는 경우에는 각각 매핑하며, 기존 테이블의 컬럼이 신 시스템의 어느 테이블로 매핑되는지는 컨버전설계서에 기입한다.
  • 테이블매핑정의서에서 기존 시스템의 테이블들은 컨버전대상목록에서 정의한 대상과 일치하여야 한다.

Impact of not having

미작성시 영향

  • 미작성시 테이블 매핑관계를 확인할 수 없어서 테이블 단위의 컨버전 설계를 수행하기 어렵다.
Reasons for not needing

작성이 불필요한 경우

  • 컨버전 대상이 없거나 스키마의 변동없이 데이터만 마이그레이션하는 경우 생략할 수 있다.
