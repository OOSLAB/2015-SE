Description
Main Description

작성항목

  1. 표준용어명 : 속성명으로 사용될 용어를 표준화하여 한글명을 기재.
  2. 표준영문명 : 실제 물리적 데이터베이스에서 컬럼명으로 사용할 영문약어명을 기재
  3. 도메인 : 해당 표준용어가 특정 도메인에 속할 경우 도메인을 기재
  4. 타입 : 데이터 형식 (해당 표준용어가 특정 도메인에 속할 경우 도메인의 데이터 형식을 기재). 해당 데이터베이스에서 지원하는 데이터 타입을 기준으로 작성
  5. 길이 : 데이터 길이 (해당 표준용어가 특정 도메인에 속할 경우 도메인의 데이터 길이를 기재)
  6. 설명 : 표준용어의 의미를 설명하고, 시스템 개발에 필요한 정보를 기술
  7. 비고 : 데이터 포맷, 참조코드 등 속성에 대해서 참조할 부가 정보를 기재

Key Considerations

작성시 고려사항

  • 표준용어명, 표준영문명의 길이가 데이터 표준 원칙에서 정의한 한글명 및 영문명의 허용 길이를 넘으면, 테이블 생성 시 오류가 날 수 있으므로 허용 길이를 넘지 않도록 정의한다.
  • 표준용어에 도메인 또는 데이터 타입(길이) 정의 시 표준용어의 특징과 맞지 않게 정의 시(예를 들어, 표준용어의 최대길이보다 적게 길이를 정의) 추후 개발 또는 운영 시 변경해야 하는 등 문제가 생길 수 있으므로, 표준용어의 특징에 맞도록 도메인 또는 데이터 타입(길이)를 정의하도록 한다.
  • 표준단어, 표준도메인을 기반으로 표준단어를 조합하여 속성의 특징에 맞는 표준도메인을 선택하여 정의하도록 한다.
  • 표준용어명 및 표준영문명의 길이가 데이터 표준 원칙에서 정의한 한글명 및 영문명의 허용 길이를 넘지 않도록 한다.
  • 표준용어는 표준용어의 의미를 파악할 수 있는 최소 길이로 정의한다.
  • 표준용어의 영문명의 길이가 문제 시 한글명을 변경하거나, 한글명을 구성하는 표준단어의 일부를 조합하여 하나의 표준 단어를 추가로 등록하여 영문명의 길이를 축약하여 사용할 수 있다.
Tailoring
Impact of not having

미작성시 영향

  • 데이터모델링에 필요한 속성들이 표준화되지 않아 데이터베이스의 무결성 및 생산성, 효율성 등에 대한 보장이 어렵다.
Reasons for not needing

작성이 불필요한 경우

  • 자료사전은 반드시 작성하도록 한다.
