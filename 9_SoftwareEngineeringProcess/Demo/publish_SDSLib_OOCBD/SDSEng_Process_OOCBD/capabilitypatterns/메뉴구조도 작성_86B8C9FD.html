Task: 메뉴구조도 작성
Purpose
기능 중심으로 화면의 메뉴를 도출하고, 사용자의 편의성을 최대화할 수 있도록 메뉴구조를 정의한다.

Main Description

수행 가이드(Task Guideline)

  1. 전체 시스템에서 메뉴 탐색 방법은 일관성이 있어야 한다.
  2. 메뉴의 순서 기준은 사용빈도와 사용순서에 의해 결정한다. 인덱스 성격의 메뉴일 경우는 가나다순을 적용한다.
  3. 화면에 기본적으로 보이는 메뉴 레벨의 범위를 결정해야 한다.
  4. 메뉴구조에 대한 너비와 깊이의 균형이 유지되어야 한다. '너비(Breadth)'는 계층의 각 단계에서 선택 가능한 분류의 수로 표현할 수 있으며,  '깊이(Depth)'는 계층의 단계를 의미한다. 너비가 넓고 깊이가 얕으면 사용자는 어떤 것을 선택할지 몰라 한참을 머뭇거린다. 또 너비가 좁고 깊이가 깊다면 사용자들은 원하는 페이지에 도달하기까지 많은 횟수를 클릭해야 한다. 그러므로 정보의 너비와 깊이를 다룰 때는 사용자의 인지 능력을 고려해야 한다.
  5. 메뉴는 중요도나 업무 흐름에 따라 결정해야 한다.
  6. 메뉴를 설계할 때는 반드시 확장성에 고려하여 메뉴영역에 충분한 여백을 남겨 두어야 한다. 시스템이 완성되어 고객사의 환경에 맞도록 Customizing할 때 메뉴를 추가하거나 삭제하는 경우가 종종 발생하므로, 이러한 메뉴의 변동사항에도 시스템이 유연하게 대처할 수 있어야 한다. 또, 버전을 Upgrade 시키거나, 메뉴명을 변경할 경우에도 여백은 반드시 필요하므로 시스템 제작 시 반드시 고려해야 한다.
  7. 메뉴명은 쉽고 직관적으로 이해할 수 있도록 작성해야 한다.
Steps
분석자는 유스케이스다이어그램, 유스케이스정의서와 화면프로토타입을 바탕으로 사용자가 시스템에서 요구하는 기능을 파악한다.
분석자는 파악한 전체 정보를 분류하고 메뉴구조의 깊이를 결정한다.
분석자는 분류된 기능을 메뉴구조에 매핑한다.
분석자는 사용자의 정보 접근 패턴과 편의성을 반영하여 메뉴구조를 조정한다.
분석자는 권한에 따라 사용 여부를 제어할 수 있도록 메뉴구조를 정의한다.
