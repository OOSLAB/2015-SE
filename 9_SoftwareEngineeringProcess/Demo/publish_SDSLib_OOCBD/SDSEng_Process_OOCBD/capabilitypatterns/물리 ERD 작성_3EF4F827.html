Main Description

[ERD 작성]

수행 가이드(Task Guideline)

  1. 컬럼의 길이는 DBMS의 제한에 맞도록 정의하며, 필요한 최대길이를 선정한다.
  2. 컬럼의 이름, 타입, 길이는 자료사전을 참조하여 일관성을 갖도록 정의한다.
  3. 컬럼들의 순서는 Not Null, 고정길이 컬럼들을 우선적으로 배치하여 물리적 공간의 사용효율을 높이도록 한다.
  4. 슈퍼-서브(Super-Sub) 타입 관계의 테이블인 경우, 데이터의 특성 및 데이터 처리 성격에 따라 하나의 테이블로 통합하거나 서브타입마다 하나의 테이블로 분리하거나 슈퍼타입과 서브타입을 각각 테이블로 변환할 수 있다.
  5. 논리 데이터 모델링에서 정규화된 테이블들을 성능을 고려하여 비정규화(De-Normalization) 작업을 통해 중복을 허용하도록 할 수도 있다. 하지만, 너무 많은 데이터의 중복을 허용하는 경우 데이터의 무결성을 보장하기 위한 오버헤드 비용이 훨씬 더 추가될 수 있으므로, 비정규화를 정규화 작업보다 우선하는 것은 주의해야 한다.
Steps
설계자는 자료사전을 참고하여 컬럼명을 정의하고 ERD(논리)를 ERD(물리)로 전환한다.
설계자는 데이터베이스 시스템에 맞게 각 컬럼의 데이터 타입을 정의한다.
데이터아키텍트(DA)는 ERD(물리)가 자료사전을 준수하는지, 각 컬럼의 데이터 타입이 데이터베이스 시스템에 맞게 정의되었는지 점검한다.
설계자는 데이터베이스 시스템의 특성 및 성능을 고려하여 물리적 데이터 모델을 튜닝한다.
