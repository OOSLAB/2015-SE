Task: 테스트계획서 작성
Purpose
테스트 유형(단위, 통합, 시스템, 인수 테스트)를 정의하고 각 유형의 테스트의 책임과 역할, 범위, 방법, 일정, 자원 등을 계획한다.

Main Description
수행 가이드(Task Guideline)
  1. 테스트 계획은 실제 프로젝트 진행 상황을 고려해서 현실적으로 가능한 수준으로 수립해야 한다.
  2. 테스트 계획 수립 시 고객의 적극적인 참여를 유도해야 한다.
  3. 각 테스트 유형별로, 즉 단위, 통합, 시스템, 인수 테스트를 모두 고려하여 각 유형에 필요한 활동을 모두 정의해야 한다.
  4. 테스트관리자와 분석자는 프로젝트계획서 및 요구사항정의서를 파악하여 테스트 항목을 식별하도록 한다.
  5. 만약 분석자가 프로젝트에서 철수한 경우라면, 설계자와 PL이 테스트관리자와 작업을 하도록 해야 한다.
  6. 단위테스트 계획 부분에서는 개발자 → PL → 제3자 테스터에 의한 절차를 추가하여, 단위 프로그램의 비즈니스 로직에 대한 테스트는 CTIP을, UI 측면에서의 테스트는 체크리스트와 해당 절차에 따른 3단계의 테스트를 수행하도록 정의하여야 한다
  7. 통합테스트 계획 단계에서는 반드시 점진적 통합테스트를 수행할 수 있도록 계획을 수립하여야 한다. 점진적 통합테스트는 통합테스트시나리오에 따라 프로그램을 개발하고, 시나리오대로 점진적인 통합테스트를 수행하는 것을 의미한다
  8. 테스트 유형별로 목적, 착수/종료기준을 정의하고, 단위/통합 테스트와 같은 기능 테스트를 위한 빌드 환경과 테스트 환경을 정의한다. 또한 시스템 테스트의 비기능적인 테스트를 위한 물리적인 환경도 정의하도록 한다.
  9. 결함관리 방안을 수립할 때에는 툴 적용 여부, 툴 적용 시 사용 방법, 결함관리 항목 등을 포함한 구체적인 방법을 제시해야 한다.
Steps
테스트관리자, 분석/설계자는 요구사항 정의서에서 테스트 항목을 식별한다.
테스트관리자는 테스트 유형을 정하고 상세 계획을 수립한다.
테스트관리자는 유형별 목적, 착수/종료 기준정의, 테스트 환경 등을 정의한다.
테스트관리자는 결함관리 방안을 수립한다.
테스트관리자는 PM의 승인을 득한다.
