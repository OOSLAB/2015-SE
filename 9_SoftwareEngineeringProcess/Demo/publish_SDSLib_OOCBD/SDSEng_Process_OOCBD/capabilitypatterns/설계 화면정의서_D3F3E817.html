Work Product (Artifact): 설계 화면정의서
본 산출물은 화면 분석 태스크와 화면 설계 태스크를 통해서 해당 수준의 정보가 추가되며 작성된다. 각 화면의 구성요소를 정의하고, 화면에서 발생하는 이벤트 및 처리로직을 작성한다. 또한 어떤 설계요소를 가지고 화면이 서버단과 연계되어 처리되는 지 정의한다.

Main Description

주요 작성항목

  1. 화면 레이아웃 : 화면에 대한 레이아웃을 나타낸다. 실제 Web 화면 레이아웃을 작성한 경우에는 이를 Capture하여 나타내고 그렇지 않은 경우에는 작성 툴로 간략히 정의한다.
  2. 데이터 구성항목 : 해당 화면을 구성하는 데이터 항목에 대해서 작성한다.
    • 항목명(한글) : 화면에 디스플레이 되는 항목의 한글명을 정의한다.
    • 컨트롤(영문) : 항목의 영문명.
    • 필수 : 해당 항목이 필수적으로 값을 가져야 하는 지 “Y/N/-“ 등으로 작성한다.
    • 수정여부 : 화면에서 값을 직접 editing 할 수 있는지 “Y/N/-“ 등으로 작성한다.
    • 설명 : 해당 항목에 대한 간략한 설명을 작성한다.
    • 비고/제약사항 : 값의 타입이나 기본값, 연관 데이터 항목 등의 내용을 작성한다.
  3. 처리 로직 : “이벤트명, 입력값/파라미터, 처리내용, 출력값/처리결과”는 분석단계에 정의하고, “시작JSP, 프리젠테이션 레이어 설계, 출력JSP”는 설계단계에 정의한다.
    • 이벤트명 : 도출된 컨트롤에서 발생하는 이벤트 명(ex:  데이터항목명 + 이벤트 핸들러)을 작성한다.
    • 입력값/파라미터 : 이벤트 발생시 전달되어야 하는 데이터를 기입한다.
    • 처리내용 : 이벤트에 대한 처리 로직을 기술한다.
    • 출력값/처리결과 : 이벤트 처리 후 화면 내 처리결과나 이동되는 화면명을 기술한다.
    • 시작JSP : 해당 이벤트를 처리하는 JSP 명을 기술한다.
    • 프리젠테이션 레이어 설계 : 해당 이벤트의 발생에 대한 Request URL, Request를 처리할 프리젠테이션 레이어의 컨트롤러 클래스, 호출되는 서버단 클래스와 함수명을 작성한다.
    • 출력JSP: 해당 결과가 출력되는 JSP 명을 기술한다.

작성스타일

  • 화면을 구성하는 데이터 항목의 명칭과 레이블을 정의한다. 예컨대 버튼 컨트롤의 레이블을 ‘검색’으로 할 것인지 ‘찾기’로 할 것인지 정한다. 또한 동일한 컨트롤을 여러 개 사용할 경우 배치 순서를 정의하여 화면 컨트롤 구성의 일관성을 기한다.
  • 데이터 항목의 경우 필수여부, 수정여부 등을 정의하고, 필요시 입출력 데이터 타입, 화면에 표시될 데이터의 길이, 최소값, 최대값, 입력제한조건 등은 비고/제약사항에 정의한다.
  • 화면에서 발생하는 이벤트의 처리내용과 입력값과 출력값을 정의한다. 여러 화면에서 공통으로 쓰이는 동일한 이벤트의 경우 이벤트명을 동일하게 기술한다.

Impact of not having

미작성시 영향

  • 개발 시 개발자가 임시방편적으로 화면을 구성하여 화면 전체의 일관성 유지가 어렵다.
  • 고객과 화면에 대한 합의가 이루어지지 않아 개발 후기에 잦은 변경요청의 원인이 된다.
  • 교육교재나 사용자매뉴얼 등의 문서 작성시 참조자료로 사용할 수 있는 문서를 확보하기 힘들다.
