Main Description

작성항목

  1. 시스템 개요 : 시스템의 개요 및 특성에 대해서 기술하며, 매뉴얼의 기본 구성에 대해 간략하게 기술한다.
    • 시스템 구성 : 개발한 새로운 시스템의 구성항목에 대해서 기술한다.
    • 메뉴구성 : 시스템의 구성 및 타시스템과의 연관성에 대해서 기술하며, 서스 시스템간의 관련성에 대해서도 기술한다.
  2. 사용지침
    • 시스템 시작방법 : 시스템의 시작(전원켜기)부터 종료(전원 내리기)까지에 대해서 기술한다.
      • 시스템 시작 (BOOTING포함)
      • 시스템 사용 화면
      • 시스템 종료
    • 용어 정의 : 시스템 사용에 따른 매뉴얼상의 용어를 정리한다.
      • 개발 용어
      • 업무 용어
    • 화면 작업 요령 : 화면 구성 표준(공통적인 기능키 작동, 초기 화면, 공통화면 등)을 기술하며, 기술하는 내용은 다음과 같다.
      • 화면의 기본기능 및 개요.
      • 작동(이동)방법
    • 사용 코드집 : 서브시스템에서 사용하는 코드값에 대해서 기술한다.
  3. 서브 시스템
    • 서브 시스템 개요 : 서브시스템의 개요를 간략하게 기술한다.
    • 서브 시스템 구성 : 서브시스템의 구성에 요소를 기술하며, 타 서브시스템과 서브시스템간의  연관성에 대해서 기술한다.
    • 서브 시스템 메뉴구성 : 서브 시스템별 메뉴 구조에 대해서 기술한다.
    • 화면 설명 : 웹 화면 설계서에서 기 작성된 각 화면의 레이아웃을 CAPTURE하며, 각 화면에 대해서 다음항목을 기술한다.
      • 화면 개요(업무설명)
      • 기능별 처리 과정 / 작업 설명
      • 작업 요령 / 처리 과정 / 기능키
      • 항목 설명
      • 필수 입력 항목 /도움 항목
      • 용어설명
    • 특정메시지

Key Considerations

작성시 고려사항

  • 고객이 알기 쉬운 용어로 작성한다.
  • 시스템 전반적인 개요를 기술한다.
  • 시스템 사용방법을 상세히 기술한다.
  • 시스템에서 사용되는 코드의 설명을 기술한다.
  • 시스템에서 제공하는 보고서의 내용 및 흐름을 기술한다.
  • 문제발생시 조치할 수 있는 방법이 기술한다.
Tailoring
Impact of not having

미작성시 영향

  • 사용자들이 시스템을 정확히 사용하지 못하거나 시스템의 일부분만을 사용하게 된다.
  • 사용자가 시스템을 능숙하게 사용하기까지 소요되는 시간이 길어질 수 있고 개발팀에게 지속적인 지원요청이 있을 수 있다.
Reasons for not needing

작성이 불필요한 경우

  • 사용자매뉴얼은 반드시 작성하도록 한다.(온라인도움말을 제공하는 경우에는 고객과 합의하여 사용자매뉴얼의 작성을 생략할 수 있다.)
Representation Options

제출시 고려사항

  • 이행단계 완료 시 고객에게 제출하고 승인을 받도록 한다.
