Main Description

작성항목

  1. 개요
    • 시스템 테스트 목적
      • 테스트를 수행하는 목적을 간략히 기술한다.
    • 책임과 역할
      • 시스템 테스트수행에 필요한 조직 및 구성원, 역할에 대하여 기술한다. 시스템 테스트와 관련된 프로젝트 팀원뿐 아니라 외부 지원조직, H/W, S/W 벤더 등도 다 포함한다.
    • 테스트 일정
      • 각 태스크 별 상세 일정에 대하여 기술한다.
  2. 테스트 환경
    • 테스트 구성도
      • 비기능적 시스템테스트가 수행되는 환경 구성을 가능하면 이미지로 표현한다.
    • H/W, S/W 구성도
      • 비기능적 시스템 테스트가 수행되는 각 구성요소들의 H/W, S/W 사양에 대하여 상세히 기술한다. 테스트 환경은 가급적 실 환경과 동일하게 가져가고 부득이한 경우 실 환경의 축소판 형태로 테스트 환경을 구축하고 실 환경과의 비교를 통해 결과값을 환산한다.
    • 테스트 Tool 구성도
      • 비기능적 시스템 테스트에 사용되는 Tool 및 사양에 대하여 상세히 기술한다. 특히 Player Agent 가 부족하지 않도록 장비 준비에 신경 쓴다.
  3. 테스트 시나리오 구성
    • 통합테스트 시나리오
      • 기능적 테스트 시나리오는 기존에 점진적으로 완성되어지던 통합테스트 시나리오와 동일하며 재사용 가능하다
    • 비기능적 테스트 대상 화면/ 테스트 시나리오
      • 앞에 기술된 대상업무 선정 기준에 의거하여 테스트 대상화면을 선정하고 이를 이용하여 실제 테스트를 수행할 시나리오를 작성한다. 이때 반드시 시스템 성능 목표치를 기술한다. 이때 성능 목표치는 단순히 3초 이내 라고 할 것이 아니고 평균 3초, 90%값 3초 등 정확한 표현을 사용한다. Active User, Concurrent User 등 사용하는 용어의 정의를 명확하게 하고, 부하량이 Active User 인지, concurrent User인지 등을 명확히 하여 추후 문제의 소지를 제거한다
    • 비기능적 테스트 부하량 산정방법
      • 시스템 테스트 시 사용되는 부하량 산정 기준에 대하여 기술한다. 부하량은 기존 시스템이 존재하는 경우 로그 분석을 통하여 산정하고 없는 경우 유사 사이트를 분석하여 산정한다.
  4. 테스트 결과 분석
    • 테스트 시나리오에 따른 테스트 결과를 기술한다.
  5. 종합의견
    • 테스트 결과 분석 내용에 대한 종합적인 의견이나 개선방향을 기술한다.
