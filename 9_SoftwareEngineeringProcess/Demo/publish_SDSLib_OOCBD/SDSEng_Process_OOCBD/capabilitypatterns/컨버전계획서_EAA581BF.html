Work Product (Artifact): 컨버전계획서
데이터 컨버전을 수행하는 대상, 환경, 절차, 일정, 조직별 책임과 역할 등의 계획을 상세히 정의한다.

Main Description

작성항목

  1. 컨버전 전략 : 데이터 컨버전을 수행하는 전략을 기술한다.
  2. 컨버전 대상 : 컨버전 대상 목록 산출물을 참조하여 컨버전 대상 테이블 또는 데이터를 명시하고, 전체 데이터가 아닌 경우에는 범위를 정의한다.
  3. 컨버전 조직 및 역할 : 데이터 컨버전과 관련된 프로젝트 내외부의 조직을 정의하고 각 조직별 역할과 책임을 정의한다.
  4. 컨버전 환경 : 컨버전을 수행하기 위한 물리적인 제반 환경(H/W, S/W, N/W)을 정의한다.
  5. 컨버전 절차 : 컨버전을 위한 데이터 추출, 변환, 적재 등의 프로세스를 정의하고, 컨버전을 위한 매핑작업, 우선순위 등을 정의한다.
  6. 컨버전 일정 : 매핑정의, 모의 컨버전, 본 컨버전 등의 일정계획을 수립한다.
  7. 데이터 검증방안 : 컨버전 후 데이터를 검증하기 위한 계획을 정의한다. 데이터검증 체크리스트를 별도로 작성하는 경우에는 체크리스트의 핵심내용을 추출하여 작성한다.
  8. 비상계획 : 컨버전 수행시의 데이터 백업 방안, 컨버전 수행 후의 비상시 복구방안을 수립한다.

Key Considerations

작성시 고려사항

  • 컨버전 계획서는 초기 작성 후에도 모의 컨버전 결과에 따라서 변경이 될 수 있는 문서이다. 모의 컨버전을 수행한 결과를 분석하여 계획에 대한 변경이 필요한 경우에는 고객과 협의하여 컨버전 계획서를 수정한다.
  • 여러 차례의 모의 컨버전에서는 해당 차수의 모의 컨버전의 제약사항에 대해 명시한다.
  • 컨버전 프로그램을 구동하는 우선순위는 계획서 작성단계에서 판단하기 어려운 경우가 대부분 이며, 컨버전 대상에서 정의한 우선순위에 따르는 것으로 한다. 다만, 여러차례 모의 컨버전을 수행하는 경우에는 수행결과를 바탕으로 컨버전 프로그램을 구동하는 순서를 정의할 수 있으며, 이를 컨버전계획서의 컨버전 절차 항목에 반영하거나 별도의 컨버전 시나리오를 작성하여 본 컨버전 시에 순서를 참조할 수 있도록 한다.
Tailoring
Impact of not having

미작성시 영향

  • 컨버전 대상을 파악하기가 어렵다.
  • 부적절한 순서로 컨버전을 수행하여 오류를 유발하거나 효율성을 떨어뜨릴 수 있다.
  • 컨버전 된 데이터의 정합성 검증이 어렵다.
Reasons for not needing

작성이 불필요한 경우

  • 컨버전이 과업범위가 아닌 경우에 생략할 수 있다.
