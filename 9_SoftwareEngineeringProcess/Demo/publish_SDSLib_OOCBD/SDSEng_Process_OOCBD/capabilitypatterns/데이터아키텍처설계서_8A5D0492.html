Main Description

작성항목

  1. 데이터 구성 뷰 : 전체 데이터에 대한 Layer 별 데이터구성도 및 Layer 별 역할
  2. 데이터 흐름 뷰 : 각 데이터구성 Layer 간 데이터 흐름 및 흐름설계 방안
  3. 데이터 통합/분산 뷰 : 각 데이터베이스 간 통합 및 분산요소에 대한 설계방안
  4. 데이터 관리 뷰 : 각 데이터구성을 관리하는 구성요소 및 관리 방안
  5. 데이터베이스 설계 방안 : 데이터모델링 절차, 논리 데이터모델링 방안, 물리 데이터베이스 설계 방안, 대용량 데이터베이스 설계 방안
  6. 데이터 표준화 방안 : 데이터 표준화 구축 방안, 데이터표준화 절차, 데이터 표준 지침
  7. 데이터 보안 방안 : Audit 속성설계, 테이블 Logging, 데이터 암호화 방안 등

Key Considerations

작성시 고려사항

  • 데이터 구성은 물리적 데이터베이스 구성을 고려하여 작성되어야 한다. 각 데이터베이스 내 주제영역은 개념아키텍처 정의단계에서 도출한 주제영역들과 일관성을 유지하여야 하며, 물리적 배치가 고려되어야 한다.
  • 데이터 흐름은 데이터 아키텍처의 각 Layer 사이에 데이터를 주고 받는 흐름을 파악하여 각 흐름에 대한 방법을 기술한다.
  • 데이터 통합/분산 방안은 물리적인 통합데이터베이스인 경우가 아닌 경우 데이터를 별도 서버에 적절하게 분산하거나, 분산된 데이터를 통합View로 제공하기 위한 방안들을 기술한다. 하나의 물리적 통합데이터베이스로 구축하는 경우 생략하거나, 논리적 분할에 의해 발생되는 이슈에 대한 방안을 기술한다.
  • 데이터 관리는 프로젝트의 구축범위 및 목적에 따라 작성 수준을 달리 할 수 있다. 예로 데이터 표준 같은 경우 메타데이터 솔루션 등을 활용하여 관리하거나, Excel 형태로 약식으로 관리할 수 있기 때문이다.
  • 데이터베이스 설계방안은 일반적인 설계Guide가 아닌 데이터베이스 설계자가 준수해야 할 원칙 및 방법들을 정의한다. 이를 통해 일관성 있는 설계가 이루어질 수 있도록 하며, 각 상세항목은 프로젝트의 사안별로 항목을 추가하여 정의할 수 있다.
  • 데이터표준화는 표준자료사전을 통한 데이터베이스 설계 개념을 확장한 것이다. 데이터베이스 설계의 품질 및 유지보수의 효율을 높일 수 있는 방법으로 데이터아키텍처 구축의 핵심활동으로 간주된다. 프로젝트의 규모에 따라 수준을 적절하게 조절하여 방안을 작성하도록 한다. 데이터표준화 수준을 간략하게 수행하고자 하는 경우 데이터 관리 부문의 표준관리 방안에 관련 내용을 모두 기술하고 데이터표준화 방안은 생략할 수 있다.
  • 데이터 보안 방안은 데이터베이스 설계 측면에서 보안 이슈를 식별하여 기술한다. DBMS 관련 보안요소는 기술인프라 아키텍처설계서에서 기술한다.
