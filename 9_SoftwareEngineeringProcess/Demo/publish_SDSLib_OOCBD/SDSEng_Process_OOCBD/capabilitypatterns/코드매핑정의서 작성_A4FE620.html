Task: 코드매핑정의서 작성
Purpose

기존 시스템의 데이터를 신 시스템으로 옮기기 위한 신구 데이터의 매핑을 테이블, 컬럼, 코드단위로 설계한다. 매핑은 컨버전 프로그램을 코딩할 수 있는 설계서 수준으로 상세하게 정의한다.

Main Description

수행 가이드(Task Guideline)

  1. 기존 시스템과 신 시스템의 매핑 순서는 데이터의 의존성에 의해 결정된다. 참조가 되는 데이터를 우선적으로 매핑하며, 일반적으로는 코드데이터에 대한 매핑을 우선적으로 실시하며 테이블매핑, 컬럼매핑(컨버전설계서 작성)의 순서로 수행한다.
  2. 테이블 매핑의 경우 신 시스템의 데이터 모델을 기준으로 매핑을 수행한다. 매핑이 완료되면 기존 시스템의 데이터를 기준으로 매핑 데이터를 정리하여 기존의 테이블 및 컬럼의 누락여부를 확인한다. 누락된 테이블, 컬럼에 대해서는 고객의 승인을 얻는다.
  3. 매핑이 불가능한 테이블, 컬럼등에 대해서는 고객과 협의하여 자료를 구축하는 방안을 수립한다. 이 경우에 데이터를 기각하거나, 초기값으로 일괄적으로 설정하거나, 수작업으로 입력하는 방법 등이 있다.
Steps
설계자는 기존 시스템, 신 시스템의 코드자료를 식별하고 각 코드에 대해 매핑을 정의한다.
설계자는 컨버전대상목록을 바탕으로 신 시스템 기준의 테이블 매핑을 정의한다.
설계자는 테이블의 각 컬럼단위로 매핑을 정의하고 신 시스템의 테이블단위로 컨버전설계서를 작성한다.
프로젝트리더(PL)은 매핑이 불가능한 테이블 및 컬럼에 대한 자료구축 방안을 수립하고 고객과 협의한다.(선택)
프로젝트리더(PL)은 기존 시스템 기준의 테이블, 컬럼 매핑을 통해서 데이터 컨버전에서 제외되는 데이터(테이블 또는 컬럼)에 대한 확인 및 고객의 승인을 득한다.(선택)
