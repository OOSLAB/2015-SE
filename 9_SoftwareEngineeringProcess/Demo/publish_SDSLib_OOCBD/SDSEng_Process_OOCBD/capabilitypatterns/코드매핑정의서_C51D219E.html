Work Product (Artifact): 코드매핑정의서
기존 시스템의 코드와 신 시스템의 코드간의 매핑을 정의하여 코드 컨버전을 수행할 수 있도록 설계한다.

Main Description

작성항목

  1. (TO-BE)시스템 : 신 시스템의 업무명. 주제영역이나 서브시스템명 등 쉽게 이해할 수 있는 명칭을 사용한다.
  2. (TO-BE)코드명 : 코드의 명칭
  3. (TO-BE)길이 : 코드값의 자리수. Byte단위로 적는다.
  4. (TO-BE)유효값 : 코드값
  5. (TO-BE)유효값명 : 코드명
  6. (AS-IS)시스템 : 기존 시스템의 명칭
  7. (AS-IS)코드명 : 기존 코드의 명칭
  8. (AS-IS)길이 : 코드값의 자리수(Byte단위)
  9. (AS-IS)유효값 : 코드값
  10. (AS-IS)유효값명 : 코드명
  11. (AS-IS)컨버전 로직 : 신규생성 코드, 매핑규칙, 1:N 매핑을 하는 경우에 대한 처리 등을 정의한다.

Key Considerations

작성시 고려사항

  • 데이터코드 목록 및 데이터코드정의서를 바탕으로 신 시스템의 코드 기준으로 작성한다.
  • 기존 시스템에는 없고 신 시스템에서 새롭게 정의되는 코드도 포함하여 작성하는 것이 매핑작업시 누락되는 것을 방지할 수 있다.
  • 단독코드로 분류된 경우에는 일반적으로 코드의 유효값 갯수가 많은 것이 일반적이므로 통합코드와는 별도의 시트로 매핑정의서를 작성한다.
