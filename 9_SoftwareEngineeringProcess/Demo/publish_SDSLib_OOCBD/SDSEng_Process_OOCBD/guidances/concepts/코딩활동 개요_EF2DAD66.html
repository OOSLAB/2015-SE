목표

(Goal)

설계 산출물에 정의된 명세에 따라 실행 가능한 소스코드를 작성하고 이에 대한 Code Inspection을 수행하며 통합빌드를 통하여 주기적으로 자동화된 빌드를 수행하고 단위 테스트를 실시하여 완전성을 검증한다

착수기준

(Entry Criteria)

종료기준

(Exit Criteria)

-    소스코드, Code Inspection 결과서, 배포파일 작성이 완료(Finalized)된 이후

실시

화면설계, 업무설계 및 데이터설계된 내용에 따라 사용자의 업무에 적합한 웹 화면과 서버 개발에 필요한 프로그램을 코딩하고, 이에 대한 Code Inspection을 수행하며, 단위테스트용 테스트 코드를 작성, 단위 테스트를 수행하여 완결성을 검증한다.

-    소스코드

-    테스트코드

-    단위테스트체크리스트[결과]

통합빌드

정기적으로 자동화된 통합빌드를 수행하고 빌드 수행결과에 대한 모니터링과 빌드수행결과의 성공/실패 여부 및 실패 내용을 통보된 메일이나 빌드 로그를 통하여 오류를 조기에 식별하고, 빌드 수행시간을 단축한다.

-    배포파일

4.        테일러링 가이드(Tailoring Guide)

4.1          단위 테스트는 선택적으로 수행할 수 있으며, 단위 테스트 체크리스트를 활용하여 별도의 단위 테스트를 위한 테스트 케이스를 도출하지 않고 화면 위주의 테스트를 수행하는 유형과 JUnit 등 테스트 프레임워크를 이용하여 클래스의 메소드 단위로 테스트 케이스를 정의하고, 테스트 코드를 작성하여 단위 테스트를 수행하는 유형이 있다.

 

5.        고려사항(Considerations)

5.1          코딩 작업 수행시에는 명명규칙, 주석표준, 에러 메시지 표준 등 개발표준을 작성하여 이를 준수하도록 하며, 형상관리툴을 이용한 버전관리 및 백업관리를 하도록 한다.

5.2          Code Inspection은 툴을 개발자 PC에 설치하여 개발자 자체적으로 상시 수행하는 상시 Code Inspection과 정기적인 일정계획 수립 후 전체 소스를 통합하여 수행하는 정기 Code Inspection으로 구분하여 수행한다.

5.3          통합빌드환경 관리자를 지정하여 통합빌드환경에 대한 모니터링을 통하여 정기적으로 자동화된 통합빌드가 지속되도록 관리한다.

5.4          통합빌드는 시스템에 의해 자동으로 수행되므로 수행절차는 시스템에 의해 수행된 결과를 모니터링하고 처리하는 절차로 구성되며, 자동화된 통합빌드환경은 CTIP 적용가이드를 참고한다.

5.5          자동화된 통합빌드환경을 위한 빌드 스크립트(Build.xml) 작성은 통합빌드환경 구성내에 포함되며 빌드파일 작성가이드를 참조하여 작성한다.
