구분

목표

(Goal)

분석 단계에서 정의한 논리데이터 모델과 자료사전을 기준으로 목표 데이터베이스 시스템에서 사용할 데이터 코드를 설계하고, 데이터베이스 시스템의 특성 및 성능을 고려하여 물리적인 데이터베이스 구조를 설계한다.

물리데이터 모델링

논리적 데이터 모델을 데이터베이스 시스템의 특성 및 성능을 고려하여 물리 데이터 모델로 전환한다.

-    테이블목록

3.        공정/산출물/역할간의 관계(Process, Deliverables & Role Relationship)

 

 

4.        테일러링 가이드(Tailoring Guide)

4.1          물리데이터 모델링 시 데이터 코드 설계내용이 반영되어야 하며, 물리데이터 모델링을 수행하면서 필요한 데이터 코드들을 도출하여 설계한다.

4.2          기존의 데이터코드 체계를 그대로 사용하며, 관련 자료가 구비된 경우(예 : 고객으로부터 관련 자료를 받은 경우)에는 데이터코드 설계 태스크는 생략할 수 있다.

 

5.        고려사항(Considerations)

5.1          물리데이터 모델링 수행 시 자료사전을 참조하여 컬럼을 정의하도록 한다.

5.2          논리데이터 모델을 물리데이터 모델로 전환 시 성능 등을 고려하여 하나의 엔티티는 여러 테이블로 분할될 수도 있으며, 여러 엔티티가 하나의 테이블로 통합될 수도 있다.

5.3          물리데이터 모델링 작업을 수행하면서 데이터 코드 테이블 설계 시 데이터 코드 설계 내용을 반영하여 코드체계와 일관성을 유지하여 코드 테이블을 설계하도록 한다.
