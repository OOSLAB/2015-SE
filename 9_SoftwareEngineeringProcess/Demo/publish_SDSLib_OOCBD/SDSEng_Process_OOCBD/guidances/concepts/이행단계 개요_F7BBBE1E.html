Concept: 이행단계 개요
Main Description

1.        개요 (Overview)

구분

내용

목표

(Goal)

테스트된 시스템을 고객이 인수한 후에 원활하게 사용할 수 있도록 필요한 작업을 수행한다.

수행 범위

(Process Scope)

l         이행계획 수립: 개발한 시스템을 고객에게 이관하기 위해서 필요한 릴리이즈 계획과 사용자 교육 훈련 계획을 수립

l         컨버전 및 릴리이즈: 개발한 시스템을 실제 운영 환경에서 운영 가능하도록 설치하고 고객의 검증을 거친 후 시스템을 고객에게 이관

l         시스템 사용교육: 개발한 시스템을 사용하고 운영, 유지보수 하는데 필요한 매뉴얼을 제공하고 시스템 사용 및 운영 방법에 대한 교육 실시

l         시범 운영: 실제 운영 환경에서의 시스템 시범운영 계획을 수립하고, 시범운영을 통하여 안정성 여부를 확인

 

2.        정책 (Policy)

2.1          프로젝트 팀은 새로운 시스템을 사용자에게 어떤 방식으로 전개 실시할 것인지에 대한 이행 계획을 수립한다.

2.2          고객의 인수 테스트가 완료된 후에 기존 운영 시스템을 신규로 전환하고 시행한다.

2.3          프로젝트 팀은 교육 계획을 검토하여 해당 교육이 효율적으로 구성되어 있는지를 확인한다. 

2.4          시범운영 이전에 사용자와 운영자의 교육이 이루어져서 이행단계 후 시스템 운영에 차질이 없도록 한다.

2.5          프로젝트 팀은 시스템을 지원하는 성능, 보안, 제어, 컴퓨터 운용 기능들에 대해 사용자 매뉴얼을 작성한다. 또한 필요 시 사용자 매뉴얼로부터 필요한 교육용 자료와 강사용 자료를 작성한다.

2.6          시범 운영을 통하여 신 시스템의 안정성 여부를 확인하여, 구 시스템에서 신 시스템으로 변경했을 경우 장애가 발생하지 않도록 한다.

2.7          시범 운영이 계획대로 성공적으로 수행되고 나면 인수책임자의 승인과 함께 프로젝트의 모든 산출물을 고객에게 전달하고 인계한다. 이때 이행 단계에서 발생한 시스템의 변경 사항이나 개선된 사항이 있는 경우에는 이를 산출물(운영에 필요한 산출물에 한함)에 반영하여야 한다.

4.        고려사항(Consideration)

4.1          고객에게 시스템을 인도할 때 위험요소를 줄이기 위하여 개발단계에서 충분한 테스트와 운영환경에 대한 성능 튜닝을 수행하도록 한다.

4.2          매뉴얼 작성 및 사용자 교육은 프로젝트 수행 시 소홀히 하기 쉬운 부분이나 고객의 만족도에 미치는 영향은 큼으로 주의하도록 한다.
