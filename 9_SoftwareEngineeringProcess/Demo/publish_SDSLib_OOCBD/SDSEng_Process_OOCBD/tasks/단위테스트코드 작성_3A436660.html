Task: 단위테스트코드 작성
Disciplines: 단위테스트모델
Purpose
화면설계, 업무설계 및 데이터설계된 내용에 따라 사용자의 업무에 적합한 웹 화면과 서버 개발에 필요한 프로그램을 코딩하고, 이에 대한 Code Inspection을 수행하며, 단위테스트용 테스트 코드를 작성, 단위 테스트를 수행하여 완결성을 검증한다.

      Main Description

      수행 가이드(Task Guideline)

      1. 코딩 작업 수행시에는 명명규칙, 주석표준, 에러 메시지 표준 등 개발표준 등을 준수하도록 하며, 형상관리툴을 이용한 버전관리 및 백업관리를 하도록 한다.
      2. CASE 툴을 통하여 생성된 모델과 소스에서 관리되는 클래스간의 목록 비교등을 통하여 가능한 모델과 소스간의 Sync.를 유지하고 가급적 개발자가 임의로 클래스를 생성하지 않도록 한다.
      3. 개발자들이 Eclipse나 그에 상응하는 IDE를 사용하는 환경을 구성하도록 하며, 이 IDE가 Dimension이나 그에 상응하는 형상관리서버와 서로 연동하여 팀작업이 가능하도록 개발환경을 구축하도록 한다.
      4. Eclipse 사용시 Source 디렉토리와 Bin Directory의 분리, 통합빌드를 고려하여 프로젝트를 서브모듈로 분리하여 부분 빌드와 전체 빌드가 가능하도록 하는 등 프로젝트를 구성하는데 빌드나 실행환경을 고려하도록 한다.
      5. 개발자가 Code Inspection 수행하기 전 표준 RuleSet를 기준으로 프로젝트에 적용할 RuleSet를 정의하고 이 Rule에 준하여 Code Inspection을 수행하고 그 결과에 대해 시정조치를 할 수 있도록 한다.
      6. Code Inspection은 툴을 개발자 PC에 설치하여 개발자 자체적으로 상시 수행하는 상시 Code Inspection과 정기적인 일정계획 수립 후 전체 소스를 통합하여 수행하는 정기 Code Inspection으로 구분하여 수행한다.
      7. 단위 테스트는 선택적으로 수행할 수 있으며, 단위 테스트 체크리스트를 활용하여 별도의 단위 테스트를 위한 테스트 케이스를 도출하지 않고 화면 위주의 테스트를 수행하는 유형과 JUnit 등 테스트 프레임워크를 이용하여 클래스의 메소드 단위로 테스트 케이스를 정의하고, 테스트 코드를 작성하여 단위 테스트를 수행하는 유형이 있다.
      8. 단위테스트체크리스트를 활용하여 단위테스트를 수행한 후 테스트 결과는 단위테스트체크리스트 산출물에 기록하며, 이는 프로젝트팀의 활용도에 따라 선택적으로 작성할 수 있다.
      9. 배치처리 업무가 있는 경우, 단위테스트의 대상으로 식별된 배치처리에 대한 단위테스트를 수행하도록 한다.
      Steps
      개발자는 프로젝트 개발표준 및 설계서를 기반으로 소스코드를 작성한다.
      개발자는 작성된 소스코드에 대해 Code Inspection 툴을 활용하여 Code Inspection을 수행한다.
      PL은 Code Inspection의 구문분석 결과에 따라 결함여부 판정 및 시정조치 계획 수립한다.
      개발자는 시정조치 계획에 따라 시정조치를 수행한다.
      개발자는 작성된 테스트 케이스를 기반으로 테스트 코드를 작성한다.(선택)
      개발자는 작성된 테스트 코드에 대한 단위 테스트를 수행한다.(선택)
      개발자는 단위테스트체크리스트를 가지고 화면을 통한 테스트를 실시한다.(선택)
      PL은 결함발생시 조치사항에 대해 개발자와 협의하고 이에대한 처리사항을 관리한다.(선택)
      개발자는 결함 발견시 코드를 수정하고 재테스트를 실시한다.(선택)
