Main Description

1. 분석 클래스다이어그램 작성 절차



2. 입력물

유스케이스 정의서 ::= 시나리오

시나리오 ::= 문장

문장 ::= 명사 + 동사 + 부사


3. 출력물

분석클래스다이어그램 ::= 클래스 + 연관 

클래스 ::= 이름 + 속성 + 오퍼레이션  + 스테레오타입

연관 ::= 이름 + 방향성 + 다중성 + 역할


4. 매핑 가이드

4.1. 분석클래스는 일반적으로 모델-뷰-컨트롤러의 아키텍처 패턴으로 분류하여 정의한다.

      - 모델은 엔티티 스테레오타입 클래스로 정의되며, 지속적 데이터의 저장 장소의 의미를 갖는다.

      - 뷰는 바운더리 스테레오타입 클래스로 정의되며, 사용자와의 상호작용을 정의한다.

      - 컨트롤러는 컨트롤 스테레오타입 클래스로 정의되며, 바운더리 클래스와 엔티티 클래스 간의 데이터 매핑 을 정의한다.

4.2. 유스케이스 정의서 시나리오의 명사를 중심으로 클래스를 추출한다.

4.3. 추출된 클래스를 엔티티, 바운더리, 컨트롤 클래스로 분류한다. 

4.4. 해당 클래스와 연관된 동사를 중심으로 연관을 추출한다.

     - 연관은 메시지와 달리 지속적, 행위적 패턴을 중심으로 정의한다. 

     - 지속적, 행위적 패턴은 반복적으로 메시지를 주고 받는 상호작용의 형태를 의미한다.
