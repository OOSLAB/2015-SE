Work Product (Artifact): 설계 화면목록
어플리케이션의 프리젠테이션 로직을 담고 있는 웹 화면(HTML / 서버페이지) 목록을 작성한다.

Main Description

작성항목

  1. 업무명: 어플리케이션 기능구조를 바탕으로 관련 업무명을 기술한다.
  2. 화면ID: 화면을 식별할 수 있는 영문ID를 정의한다.
  3. 화면명: 해당 화면의 성격과 기능을 기술한 것으로 한글로 기재한다.
  4. 유스케이스ID: 화면이 정의된 관련 유스케이스ID를 기술한다.
  5. 유형: 해당 화면의 레이아웃 유형을 기술한다.
  6. 비고: 참조할만한 간략한 부가 설명을 기술한다.

Key Considerations

작성시 고려사항

  • 화면명 : 프로젝트 명명규칙을 따르고, 적절하게 부여한다.
  • 작성 범위 : 유스케이스를 구현하기 위한 모든 화면을 빠짐없이 정의한다.
Tailoring
Impact of not having

미작성시 영향

  • 도출된 전체 화면의 현황을 파악하기 어려워 프로젝트 관리에 문제가 발생한다.
Reasons for not needing

작성이 불필요한 경우

  • 화면목록은 반드시 작성하도록 한다.
Representation Options

제출시 고려사항

  • 설계단계 완료 시 고객에게 제출하고 승인을 받도록 한다.
