Work Product (Artifact): 데이터코드정의서
효율적인 데이터 처리를 위해 코드화하여 관리할 항목들에 대해 각 코드별로 코드체계를 바탕으로 코드ID, 코드명 등을 문서화하여 작성한다.

Main Description

작성항목

  1. 시스템명 : 데이터 코드 목록의 시스템명과 동일
  2. 코드분류ID : 코드 분류 ID. 데이터 코드 목록의 코드분류ID와 동일
  3. 코드분류명 : 코드 분류 명칭. 데이터 코드 목록의 코드분류명과 동일
    • 예) 지역코드, 업무구분코드 등
  4. 코드ID : 코드 분류에 포함되는 실제 코드 값
  5. 코드명 : 코드ID에 대응하는 명칭
  6. 비고 : 코드의 레벨/구분 등 부가적인 정보
  7. 설명 : 코드명만으로 이해하기 힘든 경우 부가적인 설명 기재

 

작성스타일

  • 통합코드
    • 통합코드는 동일한 형태로 관리되므로 하나의 Sheet로 통합하여 작성하고, Sheet명을 ‘통합코드’로 한다.
  • 단독코드
    • 단독코드는 각자 관리하는 체계가 다르므로 각 코드별로 하나의 Sheet로 작성하고, Sheet명을 코드분류명으로 한다.

Key Considerations

작성시 고려사항

  • 데이터코드목록에 정의한 코드 체계 및 길이에 맞추어 코드ID를 정의하도록 한다.
  • 통합코드와 단독코드를 구분하여 정의하도록 한다.
  • 코드ID, 코드명이 중복 정의되지 않도록 한다.
  • 각 업무별로 단독으로 사용되지 않고 모든 업무에 공통적으로 사용되는 코드에 대해서는 시스템명을 별도로 분리하여 작성하도록 한다. (예를 들어, 시스템명을 ‘공통’으로 분류하여 공통코드들을 정의)
Tailoring
Impact of not having

미작성시 영향

  • 데이터코드가 명확히 정의되지 않아 데이터의 일관성 유지가 어렵고 코딩의 효율성이 떨어지게 된다.
Reasons for not needing

작성이 불필요한 경우

  • 데이터코드정의서는 반드시 작성하도록 한다.

제출시 고려사항
