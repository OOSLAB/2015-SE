Task: 데이터코드목록 작성
Purpose
코드화하여 관리할 필요가 있는 데이터 항목들을 추출하고, 각 항목들에 대해 코드체계를 설계한다.

Main Description

수행 가이드(Task Guideline)

  1. 데이터아키텍처설계서에서 표준으로 정의한 코드 도메인을 참조하여 코드를 설계한다.
  2. 업무별로 같은 코드를 다른 코드로 중복 설계하지 않도록 한다.
  3. 가능한 범위의 데이터 항목들에 대해 모두 코드화하여 관리하도록 한다.
  4. 코드의 성격에 따라 통합코드와 단독코드로 분류하여 설계하도록 한다. 통합코드는 코드/코드 값만 관리하는 수십 개 이내의 코드에 해당되며, 하나의 테이블에서 여러 개의 코드를 관리할 수 있다. 단독코드는 독립적인 별도의 테이블을 생성하여 관리해야 하는 코드에 해당되며 일반적으로 코드/코드 값 이외의 추가적인 관리 속성이 존재한다.
  5. 향후 확장성을 고려하여 코드의 길이를 정의한다.
Steps
설계자는 ERD(논리)에서 코드화하여 관리할 필요가 있는 데이터 항목들을 추출한다.
설계자는 코드의 성격을 분석하여 통합코드와 개별 테이블로 관리할 단독코드로 분류한다.
설계자는 각 코드별로 코드체계를 설계한다.
데이터아키텍트(DA)는 설계자들이 설계한 코드체계가 데이터아키텍처설계서에서 정의한 표준코드도메인을 준수하는지 점검한다.
