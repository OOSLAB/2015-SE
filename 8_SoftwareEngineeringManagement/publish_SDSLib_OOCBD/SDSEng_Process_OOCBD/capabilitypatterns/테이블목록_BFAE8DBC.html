Work Product (Artifact): 테이블목록
물리적 데이터 모델링 단계에서 도출된 데이터베이스로 관리될 테이블들에 대한 목록을 문서화한다.

Main Description

작성항목

  1. 시스템명
  2. 주제영역명 : 논리데이터 모델링 및 물리데이터 모델링 시 분류한 주제영역
  3. 테이블ID : DBMS에서 구현되는 물리적인 테이블 명칭.
  4. 테이블명 : DBMS에서 물리적으로 구현되는 테이블에 대한 한글명(일반적으로 ERD(논리)의 엔티티명임).
  5. 길이 : 하나의 Row에 대한 총길이의 평균값을 바이트수로 기재
  6. 초기건수 : 테이블 최초 생성시의 건수
  7. 최대건수 : 시스템 수명을 고려한 테이블에서 저장해야 할 최대 건수
  8. 증가건수 : 테이블의 년단위 증가 건수

Key Considerations

작성시 고려사항

  • 테이블ID의 길이가 너무 길게 정의되면 테이블 생성 시 오류가 생길 수 있으므로 데이터베이스가 허용하는 길이를 넘지 않도록 정의한다.
  • 테이블ID는 데이터베이스가 허용하는 길이를 넘지 않고, 중복되는 테이블ID가 없도록 한다.
  • 시스템에서 관리되는 테이블들을 모두 정의하고, 테이블정의서에 정의한 테이블들과 일관성이 맞도록 정의하도록 한다.
