Description
Main Description

작성항목

  1. 테스터 : 본 결함보고서를 작성한 테스터를 기재한다
  2. 시나리오ID: 테스트를 수행한 시나리오ID를 기록한다
  3. 작성일 : 산출물 작성일자를 기재한다.
  4. 승인자 : 프로젝트관리자가 조치결과를 승인한 서명을 기재한다.
  5. 결함ID : 결함내용별로 부여하는 ID를 기재한다. 표준 및 절차매뉴얼에 정의된 ID 명명규칙에 준하여 ID를 기재한다. 툴을 사용하는 경우 툴에서 제공하는 ID를 사용해도 좋다
  6. 케이스ID : 결함이 발생된 케이스의 ID를 기재한다
  7. 결함내용 : 테스트중 발생한 결함내용을 상세히 기재한다. 조치자가 결함내용을 명확히 이해할 수 있을 정도로 상세히 기술한다. 테스트중 결함이 발생한 클래스 ID, 클래스명을 함께 기재한다. 여백이 부족한 경우 별첨으로 별도의 무형식 문서를 첨부할 수 있다.
  8. 심각도 : 결함내용에 따라 결함의 심각도를 입력한다. 심각도의 종류는 치명결함, 중결함, 경결함, 단순결함, 권고사항, 요구사항으로 이루어 진다. 테스트계획서에 정의된 심각도 판정방법을 참조하여 기재한다.
  9. 조치자 : 결함을 직접 수정한 조치자를 기재한다
  10. 상태 : 결함 해결에 대한 현재 상태를 기재한다.
  11. 결함발생시점 : 결함이 발생된 단계 기재한다. 결함을 발견한 시점이 아닌 결함의 원인이 되는 부분이 심어신 단계를 의미한다. 단계는 분석, 설계, 개발로 구분되며 상세한 내용은 테스트계획서에 정의된다
  12. 완료일 : 결함이 조치 및 재테스트까지 완료된 최종 확인 일자를 기재한다

Key Considerations

작성시 고려사항

  • 테스트 수행 중 발견된 결함에 대해 결함목록을 작성한다.
  • 결함이 발생한 결함ID를 기재한다.
  • 결함이 발생한 내용과 조치결과를 기재한다.
