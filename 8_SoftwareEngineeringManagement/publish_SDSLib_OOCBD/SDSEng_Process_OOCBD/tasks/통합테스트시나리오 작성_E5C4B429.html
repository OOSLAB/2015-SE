Task: 통합테스트시나리오 작성
Disciplines: 통합테스트모델
Purpose
시스템의 기능적인 테스트를 위해 유스케이스 기반의 테스트 케이스를 도출하여 통합테스트 시나리오를 작성한다.

      Process Usage
      • 통합 테스트  >  통합테스트시나리오 작성
      Main Description
      수행 가이드(Task Guideline)
      1. 테스트 케이스는 요구사항을 만족할 수 있도록 작성이 되어야 한다. 테스트 데이터의 조합을 통해 케이스를 다양하게 변경할 수 있으므로, 다양한 테스트 데이터를 대입할 수 있도록 한다.
      2. 테스트 계획 수립 시 고객의 적극적인 참여를 유도해야 한다.
      3. 입력 산출물인 테스트계획서, 요구사항정의서, 화면정의서, 유스케이스정의서를 충분히 검토한 후 통합테스트 설계 준비를 한다.
      4. 각 산출물로부터 통합테스트에 필요한 항목을 도출한다
      5. 통합테스트 시나리오는 유스케이스를 기준으로 작성한다. 따라서 통합테스트를 실시하기 전에 통합테스트시나리오에 유스케이스의 플로우를 반영하고, 통합테스트 실시를 위한 테스트데이터, 예상결과등을 기록한다.
      6. 여러 개의 유스케이스들의 조합으로 구성되는 비즈니스 흐름에 대한 테스트시나리오는 유스케이스 크기 수준으로 정의된 액티비티의 흐름을 나타내는 액티비티다이어그램에서 정의된 비즈니스 프로세스 수준으로 작성한다.
      7. 작성된 통합테스트 시나리오는 점진적 통합테스트가 이루어질 수 있도록 작성이 되었는지를 최종 검증하여야 한다. 즉, 개발 순서가 통합이 용이하도록 되어있고, 통합빌드를 수행하면서 점진적으로 프로그램의 단위를 키워나가는 테스트가 이루어질 수 있도록 작성되었는지를 검증하여야 한다.
      8. 통합테스트 시나리오에 대해서 분석자의 최종 검증을 득하도록 하며, 시나리오는 지속적으로 변경이 될 수 있기 때문에, 테스트관리자의 관리하에 형상관리를 수행하도록 한다.
      Steps
      분석자, PL은 입력 산출물을 검토하여 통합테스트시나리오 작성을 준비한다.
      분석자, PL은 유스케이스정의서로부터 유스케이스의 흐름을 반영하여 테스트케이스 도출한다.
      분석자, PL, 테스트관리자는 유스케이스와 매핑이되는 테스트시나리오를 작성한다.
