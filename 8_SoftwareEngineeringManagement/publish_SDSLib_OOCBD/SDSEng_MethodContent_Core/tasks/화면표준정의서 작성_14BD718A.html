Task: 화면표준정의서 작성
Disciplines: 분석 화면모델
Purpose
화면레이아웃 정의, 메뉴구조 정의, 네비게이션 정의 시 준수해야 할 표준을 정의한다.  색상, 프레임, 화면 구성 요소의 배치 스타일 및 이미지 등 화면 개발 시 필요한 디자인적 요소들을 정의한다.

    Main Description

    수행 가이드(Task Guideline)

    1. 화면 레이아웃 종류 식별 : 화면 프로토타입을 참조하거나 업무 내용상으로 어떤 종류의 화면이 필요한지를 식별한다. 예를 들어 목록조회 화면, 탭이 있는 화면, 상세조회 화면이면서 정보가 그룹별로 묶여 있는 화면 등 다양한 형태의 화면 유형을 들 수 있다.
    2. 화면 디자인 제약사항 식별 : 고객사의 CI(Corporate Identity) 정책에 따른 색상 프로파일/폰트 등이 제한, 디스플레이 기기의 제약으로 인한 해상도 제한, 네트워크 속도 문제로 인한 이미지 파일 크기 제한 등 각 고객에 특화된 사유로 인한 화면 디자인에서의 제한 요소들을 파악해야 한다.
    3. 화면 네비게이션 스타일 정의
      1. 업무 프로세스 스타일에 따라 :  기본적인 CRUD (등록, 조회, 수정, 삭제) 작업의 흐름에 따른 네비게이션 스타일, 그 외 업무 프로세스에 따라 달라지는 네비게이션 스타일을 정의한다. 예를 들어 일괄 변경 업무와 특정 대상을 선택하여 변경하는 업무는 화면 네비게이션이 다르게 정의된다.
      2. 이벤트 결과처리 방식에 따라 :  어떤 작업 후에 화면이 변경되도록 구성된 화면과, 변경되지 않고 해당 화면이 다시 출력 되도록 구성된 화면 등 화면 구성 스타일에 따라서도 네비게이션 스타일을 정의할 수 있다. 예를 들어 여러 개의 탭으로 구성된 화면에서 각 탭별로 개별 등록 작업을 수행할 수 있고, 개별 등록 작업이 수행된 후에는 다른 화면으로 이동하지 않고 등록 성공 메시지만 나오는 경우와, 여러 탭의 내용을 한꺼번에 등록하고 등록 후에는 등록 결과 조회 화면으로 이동하는 경우를 들 수 있다.
    4. 이벤트 표준 정의 :  공통적으로 사용하는 화면이나 화면 구성요소에서 발생하는 이벤트명과 입/출력값 등을 표준화하여 정의한다. 예를 들어 ‘고객번호조회’ 이벤트의 경우 입력값은 주민번호, 고객ID이고 출력값은 고객번호로 정의한다.
    Steps
    분석자는 화면 레이아웃 정의에 필요한 업무 유형을 정의한다.
    UI기획자는 업무 유형별 화면 레이아웃을 정의한다.
    UI기획자는 화면의 시각적 요소를 제약사항을 반영하여 디자인한다. (색상 프로파일, 프레임 배치, 각 레이아웃 종류별 구성 요소 배치, 공통 사용 이미지, 폰트 등)
    UI기획자는 이벤트 표준을 정의한다.
    UI기획자는 메뉴구조의 깊이를 결정한다.
    UI기획자는 화면 네비게이션 스타일을 정의한다.
    분석자는 화면표준을 업무적 관점에서 검토한다.
    PL은 화면표준에 대해 고객 승인을 받는다.
