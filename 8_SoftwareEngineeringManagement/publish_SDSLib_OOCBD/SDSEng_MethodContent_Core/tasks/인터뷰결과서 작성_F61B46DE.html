Task: 인터뷰결과서 작성
Disciplines: 인터뷰모델
Purpose
사용자와의 인터뷰를 통해 현행분석 결과를 확인하고 추가적인 문제점 및 개선요구사항을 도출한다.

      Process Usage
      • 인터뷰 수행  >  인터뷰결과서 작성
      Main Description

      수행 가이드(Task Guideline)

       

      1. 현행업무흐름도에서 담당업무별로 정의된 내용 중 보다 상세한 정보가 필요한 경우 해당 업무 담당자와의 인터뷰를 통해 이를 확인한다.
      2. 여러 사용자 부서의 의견을 취합하여 조정하는 것이 중요한 경우에는 개별적인 인터뷰와 병행하여 관련 사용자 그룹과 워크샵을 실시한다.
      3. 인터뷰 시 질문사항은 간결하고 명확하게 한다.
      4. 인터뷰를 실시할 고객명(고객 담당자)과 질문내용을 사전에 준비한 후 고객과 인터뷰를 실시한다.
      5. 사전에 준비한 인터뷰 내용 외에 추가적인 질문내용이 있으면 해당 질문내용과 고객담당자의 응답을 인터뷰 결과에 같이 기록한다.
      Steps
      현행분석 결과를 검토하여 분석자가 이해할 수 없거나, 보다 명확하고 상세한 정보가 요구되는 사항을 도출한다.
      인터뷰 계획을 수립한다.
      • 인터뷰 대상자 선정
      • 인터뷰 수행팀 구성
      • 인터뷰 일정 수립
      • 인터뷰 내용 정리(인터뷰 대상자별로)
      인터뷰를 실시한다.
      • 현행분석 결과 확인
      • 현행분석 시 누락된 추가적인 요구사항 도출
      인터뷰 결과를 정리한다.
