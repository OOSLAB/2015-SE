Task: 컨버전대상목록 작성
Disciplines: 비기능요구모델
Purpose
기존 시스템의 데이터를 기준으로 컨버전 대상이 되는 데이터를 선정하여 과업범위를 확정하며, 컨버전 우선순위를 정한다.

      Main Description

      수행 가이드(Task Guideline)

      1. 컨버전의 과업범위는 업무의 과업범위와 마찬가지로 요구정의 단계에서 정의해야 하는 중요한 결정 요소이므로, 고객과의 협의를 통해서 정의해야 한다.
      2. 모든 데이터를 오픈전에 하는 본 컨버전에서 컨버전해야 하는 것은 아니다. 데이터의 성격에 따라서 사전 컨버전 또는 사후 컨버전할 수 있다. 사전 또는 사후 컨버전이 가능한 데이터를 식별하고 고객과 협의한다.
      3. 컨버전의 우선순위는 데이터의 중요도 및 컨버전 의존관계를 고려하여 정한다. 데이터의 정합성 체크를 위해서는 참조되는 데이터가 우선적으로 컨버전되어야 한다.
      4. 컨버전 대상이 되는 데이터량이 주어진 컨버전 시간에 비해서 많은 경우에는 업무에 필요한 최근 데이터만 우선순위를 두어 컨버전하고, 오픈 후에 나머지 데이터를 컨버전 할 수 있다. 업무에 직접적으로 필요하지 않은 과거의 데이터나 월말 작업등을 위해 필요한 데이터들이 대상이 된다.
      5. 기존 시스템의 테이블 또는 컬럼 중 컨버전 대상에서 제외되는 경우에는 반드시 고객과 협의해야 하며, 컨버전 대상 선정을 기존 시스템을 기준으로 함으로써 컨버전 대상에서 제외되는 데이터를 쉽게 확인할 수 있다.
      6. 신 시스템의 데이터 모델이 확정된 후에 컨버전 대상선정을 진행하게 되면 기존 데이터의 정합성 확인 및 오류데이터를 수정할 수 있는 시간이 상대적으로 부족하므로  컨버전 데이터의 품질을 확보하기 위해서는 분석단계에서 기존 시스템의 데이터 모델을 기준으로 수행한다.
      Steps
      분석자는 현행시스템분석서를 바탕으로 기존 시스템의 데이터목록(테이블, 파일 등)을 작성한다.
      분석자는 임시 데이터로 전환대상이 아닌 자료를 우선적으로 식별한다.
      분석자는 데이터의 성격을 분류한다. 일반적으로는 코드, 업무, 로그성 데이터로 분류한다.
      분석자는 컨버전 대상이 되는 데이터를 고객과 함께 식별한다.
      분석자는 컨버전 우선 순위를 정한다. 일반적으로 코드, 업무, 로그 데이터 순으로 컨버전 우선 순위를 정한다.
