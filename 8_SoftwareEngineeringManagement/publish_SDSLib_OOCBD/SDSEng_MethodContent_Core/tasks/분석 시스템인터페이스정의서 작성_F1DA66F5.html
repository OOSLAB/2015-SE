      Main Description

      수행 가이드(Task Guideline)

       

      1. 시스템인터페이스는 시스템인터페이스 요구사항과 유스케이스다이어그램의 유스케이스와 부액터간의 관계를 통해 식별한다.
      2. 시스템인터페이스는 유스케이스 분석 산출물과 일관성을 확보해야 한다.
        • 시퀀스다이어그램이 작성된 경우, 시스템인터페이스목록은 시스템인터페이스를 담당하는 바운더리클래스가 받는 메시지와 일관되어야 한다.
        • 클래스다이어그램(분석)의 시스템인터페이스를 담당하는 바운더리클래스에 정의된 오퍼레이션과 일관되어야 한다.
      3. 연계 대상 시스템은 개념아키텍처에 정의된 목표시스템구성도 혹은 컨텍스트다이어그램의 연계 대상 범위에 포함되는지 확인하고, 필요한 경우 고객과 협의하여 결정한다.
      4. 연계 대상 시스템이 대외시스템인 경우, 대외기관의 연계표준을 미리 확인하여야 한다.
      5. 처리 유형 및 주기는 업무 특성과 데이터 특성, 성능 등을 고려하여 정의한다.
        • 실시간 처리 유형은 사용자가 시스템을 통해 업무 수행 중 바로 연계해야 할 경우에 해당된다. 조회 성격의 업무나 사용자가 바로 확인해야 하거나 응답시간이 길지 않는 경우 주로 적용한다.
        • 배치 처리 유형은 대량의 데이터를 주고 받거나 사용자가 바로 확인할 필요가 없는 업무에 주로 적용한다.
        • 지연 처리 유형은 연계 시 발생하는 비용이 클 경우, 일정 Term을 두고 데이터를 모았다가 연계하는 경우 해당된다. 주로 대외기관과 연계시 적용한다.
        • 배치와 지연처리 유형에 대해서는 주기를 구체적으로 정의한다. 정의시 데이터량 및 성능을 고려한다.
      6. 시스템인터페이스가 처리 성격일 경우에는 Target시스템에서의 처리 로직을 정의하여 기능요건에 포함시킨다.
      7. 데이터항목은 모든 인터페이스에 공통 적용되는 데이터항목을 고려하여 정의하며, 양방향 통신인 경우에는 요청 시의 데이터항목과 응답 시의 데이터항목을 구분하여 정의한다.
      Steps
      분석자는 시스템인터페이스 요구사항 및 유스케이스다이어그램을 기반으로 시스템인터페이스 목록을 도출한다.
      분석자는 시스템인터페이스와 관련된 Source시스템과 Target시스템을 정의한다.
      분석자는 시스템인터페이스를 시스템별로 분류하고 시스템 연계 대상 범위를 확인하여, 필요시 고객과 협의한다.
      분석자는 시스템인터페이스의 업무 및 데이터 특성, 성능을 고려하여 처리유형 및 주기를 정의한다.
      분석자는 시스템인터페이스의 기능 요건을 정의한다.
      분석자는 시스템인터페이스의 입출력 데이터항목을 정의한다.
      분석자는 대상 시스템의 담당자와 시스템인터페이스 정의 내용을 검토한다.
