      Process Usage
      • 배포 준비  >  배포정의서 작성
      Main Description

      수행 가이드(Task Guideline)

      1. 업무특성과 프로젝트 특성을 고려하여 배포단위 기준을 정의하는데, 배포단위는 시스템 패치작업이나 배포파일간의 종속성에 영향을 미친다. 물리적인 배포파일에 포함되는 파일구조를 정의한다.
      2. 배포단위와 배포구조정의에 맞게 배포정의서를 작성한다. 컴포넌트를 기준으로 하는경우 EJB-Descriptor 같은 컴포넌트 구성요소도 고려해야 한다.
      Steps
      SA는 분석패키지다이어그램 기준으로 업무정의에 대한 기준을 제시한다.
      SA는 배포파일의 관리를 고려하여 그룹을 정의하고 배포단위를 결정을 위한 기준을 제시한다.
      설계자는 배포파일에 포함될 소스파일과 설정파일을 정의한다.
      설계자는 배포정의서를 작성한다.
