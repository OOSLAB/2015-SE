Description
Main Description

작성항목

  1. 테이블 ID : 신 시스템의 테이블ID
  2. 테이블명 : 신 시스템의 한글 테이블명
  3. 업무구분 : 신 시스템의 업무명 또는 서브시스템명.
  4. Target : 신 시스템의 테이블ID, 컬럼ID, 데이터타입, 길이 등을 명시한다.
  5. Source : 기존 시스템의 테이블ID, 컬럼ID, 데이터타입, 길이 등을 명시한다.
  6. 매핑룰 : 컬럼단위의 컨버전 로직을 기술한다. 데이터에 대한 변경, 매핑 없이 컨버전 하는 경우에는 생략할 수 있다.
  7. 비고 : 테이블단위의 컨버전 로직이 필요한 경우에 명시한다. 또는 기존 시스템의 테이블 컬럼 중 컨버전 대상에서 제외되는 경우에 내용을 기술한다.

Key Considerations

작성시 고려사항

  • 신 시스템의 테이블 레이아웃을 기준으로 기존 시스템의 테이블 컬럼을 매핑하는 형식으로 작성한다. 기존 시스템의 테이블 컬럼 중 컨버전 대상에서 제외되는 경우에는 하단의 비고란에 명시하고 고객의 승인을 득한다.
  • 컨버전 로직이 아주 복잡한 경우가 아닌 이상은 별도의 컨버전 프로그램 사양서를 작성하지 않는 만큼 컨버전설계서를 바탕으로 코딩이 가능할 정도로 상세히 작성한다.
  • 매핑룰은 가급적 컨버전설계서의 매핑룰에 모두 작성하며, 부득이하게 코드매핑룰 이나 데이터 정비룰의 양이 많은 경우에는 별도 문서에 대한 참조를 할 수 있도록 명시한다.
  • 기존 시스템의 데이터 전체를 컨버전하는 것이 아니라 특정한 조건을 만족하는 데이터에 한해서 컨버전하는 경우에는 비고란에 컨버전 조건을 기술한다.
Tailoring
Impact of not having

미작성시 영향

  • 컨버전 프로그램의 처리로직이 상세화되지 못하며 이로 인하여 설계자와 개발자간에 문서가 아닌 구두로 의사소통을 하게 되므로 결함발생의 위험이 높다.
Reasons for not needing

작성이 불필요한 경우

  • 컨버전 대상이 없거나 스키마의 변동 없이 데이터만 마이그레이션하는 경우 생략할 수 있다.
