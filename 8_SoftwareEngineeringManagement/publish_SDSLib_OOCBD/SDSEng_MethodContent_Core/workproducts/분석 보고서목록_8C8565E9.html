Description
Main Description

작성항목

  1. 업무명 : 관련 업무명을 기재한다.
  2. 보고서명 : 해당 보고서를 식별할 수 있는 이름을 한글로 기재한다.
  3. 화면ID : 화면을 통해 보고서를 출력하는 경우 해당 화면ID를 기재한다.
  4. 건수/주기 : 보고서가 발행되는 주기와 해당 주기별로 발행되는 횟수를 기재한다.
  5. 주사용자 : 해당 보고서를 주로 많이 사용하는 사용자의 유형을 기재한다.
  6. 기밀구분 : 해당 보고서의 기밀구분(보안등급)을 고객의 기준에 의해 적절히 구분하여 기재한다.

Key Considerations

작성시 고려사항

  • RFP, 제안서, 계약서, 프로젝트계획서 등을 검토하여 과업범위를 명확히 한 후 기능/비기능적 요구사항이 누락되지 않도록 작성한다.
  • 보고서명 : 프로젝트 명명규칙을 따르고, 적절하게 부여한다.
  • 작성 범위 :  모든 보고서를 빠짐없이 정의한다.
  • 화면 ID : 화면목록에 정의된 화면ID와 동일해야 한다. 배치작업으로 보고서를 출력할 경우는 기재하지 않는다.
